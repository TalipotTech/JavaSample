Configure Git

One-time setup on your computer

‚öôÔ∏è Set Your Identity

Open a terminal in VS Code (Ctrl + ` or Terminal ‚Üí New Terminal) and configure your Git identity:

Terminal 
# Set your name
git config --global user.name "Your Name"

# Set your email (use the same email as your GitHub account)
git config --global user.email "your.email@example.com"

# Verify your configuration
git config --list
üí°

Why This Matters

Git uses this information to identify who made each commit. Make sure to use the same email address as your GitHub account for proper attribution.

Set up your remote repository

üåê Create New Repository

Follow these steps on GitHub:

  1. Go to github.com and log in
  2. Click the + icon in the top-right corner ‚Üí New repository
  3. Fill in the repository details:
    • Repository name: java-tutorial-project
    • Description: (optional) "Java programming tutorial samples"
    • Visibility: Public or Private
  4. Click Create repository
  5. Keep this page open - you'll need the repository URL

Initialize Git in Your Project

Set up version control locally

üìÅ Navigate and Initialize

In VS Code terminal, navigate to your project folder and initialize Git:

Terminal 
# Navigate to your project folder (if not already there)
cd path/to/your/java-project

# Initialize a new Git repository
git init

# You should see: Initialized empty Git repository in .../
üìù

What This Does

The git init command creates a hidden .git folder in your project, which Git uses to track all changes to your files.

Connect to GitHub and Push

Upload your code to the remote repository

üöÄ Push to Remote

Connect your local repository to GitHub and push your code:

Terminal 
# Add the remote repository (replace with YOUR repository URL)
git remote add origin https://github.com/YOUR-USERNAME/java-tutorial-project.git

# Rename branch to 'main' (if not already)
git branch -M main

# Push to GitHub
git push -u origin main
üìù

The -u Flag

The -u (or --set-upstream) flag links your local branch to the remote branch. After this, you can simply use git push for future pushes.

7

Authentication

Securely connect to your GitHub account

üîê Authentication Options

When you push, GitHub will ask for authentication. Choose one of these methods:

üîë Option A: Personal Access Token

  1. Go to GitHub ‚Üí Settings ‚Üí Developer settings ‚Üí Personal access tokens ‚Üí Tokens (classic)
  2. Click Generate new token (classic)
  3. Give it a name, set expiration, check the repo scope
  4. Click Generate token
  5. Copy the token (you won't see it again!)
  6. Use this token as your password when prompted

üíª Option B: GitHub CLI

Terminal 
# Install GitHub CLI first, then run:
gh auth login

# Follow the interactive prompts

üîí Option C: SSH Key

Terminal 
# Generate SSH key
ssh-keygen -t ed25519 -C "your.email@example.com"

# Start SSH agent
eval "$(ssh-agent -s)"

# Add key to agent
ssh-add ~/.ssh/id_ed25519

# Copy public key (add this to GitHub)
cat ~/.ssh/id_ed25519.pub

Then add the key to GitHub: Settings ‚Üí SSH and GPG keys ‚Üí New SSH key

Quick Reference

Common Git commands you'll use daily

‚å®Ô∏è Essential Git Commands

Command Description

git add filename Stage a specific file

Troubleshooting

Common issues and how to fix them

üõ†Ô∏è Common Problems & Solutions

"remote origin already exists"

Solution:
git remote remove origin
git remote add origin https://github.com/YOUR-USERNAME/repo-name.git

"failed to push some refs"

Solution:
git pull --rebase origin main
git push

"Authentication failed"

Solution:
  • Make sure you're using a Personal Access Token, not your GitHub password
  • Check that your token has the repo permission enabled
  • Ensure the token hasn't expired

"Permission denied (publickey)"

Solution:
  • Your SSH key isn't set up correctly
  • Try using HTTPS URL instead of SSH
  • Or regenerate and add your SSH key to GitHub
