3.1

Classes vs Objects

Understand the relationship between blueprints (classes) and instances (objects).

üèóÔ∏è What is a Class?

A class is a blueprint or template that defines the structure and behavior of objects. It specifies what data (fields) an object will hold and what actions (methods) it can perform.

Think of a class like an architectural blueprint for a house. The blueprint defines the structure, rooms, and features, but it's not a house itself - it's the plan used to build actual houses.

üìã

Class

A template that defines fields (data) and methods (behavior). Created once, used to make many objects.

üì¶

Object

An instance of a class. Each object has its own copy of the instance fields with unique values.

üî§

Fields

Variables that store the state/data of an object. Also called instance variables or attributes.

‚ö°

Methods

Functions defined in a class that describe what actions an object can perform.

üöó Creating a Class

Let's create a Car class to understand the anatomy of a Java class:

Car.java 
public class Car {
    // Fields (instance variables) - the data
    String brand;
    String model;
    String color;
    int year;
    int speed;
    boolean isRunning;
    
    // Methods - the behavior
    void start() {
        isRunning = true;
        System.out.println(brand + " " + model + " is now running!");
    }
    
    void stop() {
        isRunning = false;
        speed = 0;
        System.out.println(brand + " " + model + " has stopped.");
    }
    
    void accelerate(int amount) {
        if (isRunning) {
            speed += amount;
            System.out.println("Accelerating... Speed is now " + speed + " mph");
        } else {
            System.out.println("Start the car first!");
        }
    }
    
    void displayInfo() {
        System.out.println("--- Car Information ---");
        System.out.println("Brand: " + brand);
        System.out.println("Model: " + model);
        System.out.println("Color: " + color);
        System.out.println("Year: " + year);
        System.out.println("Speed: " + speed + " mph");
        System.out.println("Running: " + isRunning);
    }
}

Calculator.java 
public class Calculator {
    
    // Method with no parameters, no return value
    public void displayWelcome() {
        System.out.println("Welcome to Calculator!");
    }
    
    // Method with parameters, no return value
    public void printSum(int a, int b) {
        int sum = a + b;
        System.out.println("Sum: " + sum);
    }
    
    // Method with parameters AND return value
    public int add(int a, int b) {
        return a + b;
    }
    
    // Method with no parameters but has return value
    public String getVersion() {
        return "Calculator v1.0";
    }
    
    // Method returning a boolean
    public boolean isPositive(int number) {
        return number > 0;
    }
    
    // Method with multiple parameters
    public double calculateAverage(double a, double b, double c) {
        return (a + b + c) / 3.0;
    }
}
CalculatorDemo.java 
public class CalculatorDemo {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        // Call method with no return
        calc.displayWelcome();
        
        // Call method with parameters, no return
        calc.printSum(10, 20);
        
        // Call method and use return value
        int result = calc.add(15, 25);
        System.out.println("Add result: " + result);
        
        // Call method that returns String
        String version = calc.getVersion();
        System.out.println(version);
        
        // Call method that returns boolean
        System.out.println("Is 5 positive? " + calc.isPositive(5));
        System.out.println("Is -3 positive? " + calc.isPositive(-3));
        
        // Call method with multiple parameters
        double avg = calc.calculateAverage(85.5, 90.0, 78.5);
        System.out.printf("Average: %.2f%n", avg);
    }
}

üîÑ Method Overloading

Method overloading allows you to define multiple methods with the same name but different parameters. Java determines which method to call based on the arguments provided.

üìù

Overloading Rules

Methods can be overloaded by: (1) different number of parameters, (2) different types of parameters, or (3) different order of parameter types. Return type alone is NOT sufficient for overloading.

MathOperations.java 
public class MathOperations {
    
    // Overloading by NUMBER of parameters
    public int add(int a, int b) {
        System.out.println("Adding 2 integers");
        return a + b;
    }
    
    public int add(int a, int b, int c) {
        System.out.println("Adding 3 integers");
        return a + b + c;
    }
    
    public int add(int a, int b, int c, int d) {
        System.out.println("Adding 4 integers");
        return a + b + c + d;
    }
    
    // Overloading by TYPE of parameters
    public double add(double a, double b) {
        System.out.println("Adding 2 doubles");
        return a + b;
    }
    
    public String add(String a, String b) {
        System.out.println("Concatenating 2 strings");
        return a + b;
    }
    
    // Overloading for calculating area
    public double calculateArea(double radius) {
        System.out.println("Calculating circle area");
        return Math.PI * radius * radius;
    }
    
    public double calculateArea(double length, double width) {
        System.out.println("Calculating rectangle area");
        return length * width;
    }
    
    public double calculateArea(double base, double height, boolean isTriangle) {
        System.out.println("Calculating triangle area");
        return 0.5 * base * height;
    }
}
OverloadingDemo.java 
public class OverloadingDemo {
    public static void main(String[] args) {
        MathOperations math = new MathOperations();
        
        System.out.println("=== Add Method Overloading ===");
        System.out.println("Result: " + math.add(5, 10));
        System.out.println("Result: " + math.add(5, 10, 15));
        System.out.println("Result: " + math.add(5, 10, 15, 20));
        System.out.println("Result: " + math.add(3.14, 2.86));
        System.out.println("Result: " + math.add("Hello, ", "World!"));
        
        System.out.println("\n=== Area Method Overloading ===");
        System.out.printf("Circle (r=5): %.2f%n", math.calculateArea(5.0));
        System.out.printf("Rectangle (4x6): %.2f%n", math.calculateArea(4.0, 6.0));
        System.out.printf("Triangle (base=8, h=5): %.2f%n", math.calculateArea(8.0, 5.0, true));
    }
}

3.3

Constructors

Initialize objects properly when they're created using constructors.

üè≠ What is a Constructor?

A constructor is a special method that is called automatically when an object is created. It's used to initialize the object's state (fields) with default or provided values.

üìõ

Same Name as Class

Constructor must have exactly the same name as the class it belongs to.

üö´

No Return Type

Constructors don't have a return type - not even void.

üîÑ

Auto-Invoked

Called automatically when you use the new keyword.

üìö

Can Be Overloaded

You can have multiple constructors with different parameters.

Constructor Syntax 
public class ClassName {
    // Default constructor (no parameters)
    public ClassName() {
        // Initialize with default values
    }
    
    // Parameterized constructor
    public ClassName(dataType param1, dataType param2) {
        // Initialize with provided values
    }
}

üìù Default Constructor

A default constructor has no parameters. If you don't define any constructor, Java provides one automatically. But if you define any constructor, the automatic one is not provided.

Student.java 
public class Student {
    String name;
    int age;
    String major;
    double gpa;
    
    // Default constructor
    public Student() {
        System.out.println("Default constructor called!");
        name = "Unknown";
        age = 18;
        major = "Undeclared";
        gpa = 0.0;
    }
    
    public void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Major: " + major);
        System.out.println("GPA: " + gpa);
    }
}

        System.out.println("Author: " + author);
        System.out.println("Pages: " + pages);

    // Getters
    public String getTitle() { return title; }
    public String getAuthor() { return author; }
    public int getPages() { return pages; }
    public double getPrice() { return price; }
}

The 'this' Keyword

Use this.fieldName when your parameter name is the same as the field name. It distinguishes between the instance variable and the parameter.

üîÄ Constructor Overloading

Like methods, constructors can be overloaded to provide different ways of initializing objects.

Employee.java 
public class Employee {
    private String name;
    private int id;
    private String department;
    private double salary;
    
    // Constructor 1: Default
    public Employee() {
        this("Unknown", 0, "Unassigned", 0.0);
        System.out.println("Default constructor used");
    }
    
    // Constructor 2: Name and ID only
    public Employee(String name, int id) {
        this(name, id, "General", 30000.0);
        System.out.println("Name+ID constructor used");
    }
    
    // Constructor 3: Name, ID, and Department
    public Employee(String name, int id, String department) {
        this(name, id, department, 35000.0);
        System.out.println("Name+ID+Dept constructor used");
    }
    
    // Constructor 4: Full constructor
    public Employee(String name, int id, String department, double salary) {
        this.name = name;
        this.id = id;
        this.department = department;
        this.salary = salary;
        System.out.println("Full constructor used");
    }
    
    public void displayInfo() {
        System.out.println("ID: " + id);
        System.out.println("Name: " + name);
        System.out.println("Department: " + department);
        System.out.printf("Salary: $%.2f%n", salary);
    }
}

The static Keyword

üè¶ Static Example: Bank Account

Let's see static members in action with a bank account example:

BankAccount.java 
public class BankAccount {
    // Static variables - shared by ALL accounts
    private static String bankName = "Java National Bank";
    private static double interestRate = 0.025;  // 2.5%
    private static int totalAccounts = 0;
    private static double totalDeposits = 0;
    
    // Static constant
    public static final double MINIMUM_BALANCE = 100.0;
    
    // Instance variables - unique to each account
    private String accountHolder;
    private int accountNumber;
    private double balance;
    
    // Constructor
    public BankAccount(String accountHolder, double initialDeposit) {
        this.accountHolder = accountHolder;
        this.balance = initialDeposit;
        
        // Update static variables
        totalAccounts++;
        this.accountNumber = 1000 + totalAccounts;
        totalDeposits += initialDeposit;
        
        System.out.println("Account #" + accountNumber + " created for " + accountHolder);
    }
    
    // Static methods - access static data
    public static String getBankName() {
        return bankName;
    }
    
    public static double getInterestRate() {
        return interestRate;
    }
    
    public static void setInterestRate(double newRate) {
        interestRate = newRate;
        System.out.println("Interest rate updated to " + (newRate * 100) + "%");
    }
    
    public static int getTotalAccounts() {
        return totalAccounts;
    }
    
    public static void displayBankInfo() {
        System.out.println("========== Bank Information ==========");
        System.out.println("Bank Name: " + bankName);
        System.out.printf("Interest Rate: %.1f%%%n", interestRate * 100);
        System.out.println("Total Accounts: " + totalAccounts);
        System.out.printf("Total Deposits: $%.2f%n", totalDeposits);
        System.out.printf("Minimum Balance: $%.2f%n", MINIMUM_BALANCE);
    }
    
    // Instance methods - access instance data
    public void deposit(double amount) {
        balance += amount;
        totalDeposits += amount;  // Can access static from instance
        System.out.printf("Deposited $%.2f. New balance: $%.2f%n", amount, balance);
    }
    
    public void withdraw(double amount) {
        if (balance - amount >= MINIMUM_BALANCE) {
            balance -= amount;
            totalDeposits -= amount;
            System.out.printf("Withdrew $%.2f. New balance: $%.2f%n", amount, balance);
        } else {
            System.out.println("Cannot withdraw. Would go below minimum balance.");
        }
    }
    
    public void addInterest() {
        double interest = balance * interestRate;
        balance += interest;
        totalDeposits += interest;
        System.out.printf("Interest $%.2f added. New balance: $%.2f%n", interest, balance);
    }
    
    public void displayAccount() {
        System.out.println("--- Account #" + accountNumber + " ---");
        System.out.println("Holder: " + accountHolder);
        System.out.printf("Balance: $%.2f%n", balance);
    }
}
BankDemo.java 
public class BankDemo {
    public static void main(String[] args) {
        // Access static members WITHOUT creating objects
        System.out.println("Welcome to " + BankAccount.getBankName());
        System.out.println("Minimum balance required: $" + BankAccount.MINIMUM_BALANCE);
        System.out.println();
        
        // Create accounts
        BankAccount acc1 = new BankAccount("Alice Johnson", 1000.0);
        BankAccount acc2 = new BankAccount("Bob Smith", 2500.0);
        BankAccount acc3 = new BankAccount("Carol Davis", 500.0);
        
        System.out.println();
        
        // Show bank-wide statistics (static method)

        acc1.deposit(500);
        acc2.withdraw(200);

        System.out.println("\n=== Interest Rate Change ===");

        System.out.println("\n=== Final Account Summary ===");
        acc1.displayAccount();
        acc2.displayAccount();
        acc3.displayAccount();
        
        System.out.println();
        BankAccount.displayBankInfo();
    }
}

Static Methods Cannot Access Instance Members

A static method cannot directly access instance variables or call instance methods. It only has access to other static members. This is because static methods can be called without creating an object!
