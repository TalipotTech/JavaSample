1.1

Introduction to Java

Understanding Java's origins, key features, and the revolutionary WORA principle

üìú A Brief History of Java

Java's story begins in the early 1990s at Sun Microsystems. What started as a project for interactive television evolved into one of the most influential programming languages in computing history.

1991

Project "Green" Begins

James Gosling, Mike Sheridan, and Patrick Naughton initiated the "Green Project" at Sun Microsystems. The language was originally called "Oak" after a tree outside Gosling's office.

1995

Java 1.0 Released

Renamed to "Java" (inspired by Java coffee), the language was officially released with the slogan "Write Once, Run Anywhere." Netscape Navigator added Java support.

2006

Java Goes Open Source

Sun Microsystems released most of Java as open-source software under the GNU General Public License, making it freely available to developers worldwide.

2010

Oracle Acquires Sun

Oracle Corporation acquired Sun Microsystems and became the steward of Java, continuing its development and releasing regular updates.

2017

6-Month Release Cycle

Java adopted a new release cadence with feature releases every 6 months, accelerating innovation while maintaining Long-Term Support (LTS) versions.

2025

Java 25 LTS Released

The latest LTS release brings compact source files, structured concurrency, and AI-ready features, marking Java's 30th anniversary.

‚≠ê Key Features of Java

Java was designed with specific goals in mind that have made it enduringly popular:

üéØ

Simple & Familiar

Java's syntax is clean and easy to learn, based on C/C++ but removing complex features like pointers and multiple inheritance.

üß©

Object-Oriented

Everything in Java is an object (except primitives). This promotes modular, reusable, and maintainable code through encapsulation, inheritance, and polymorphism.

üåç

Platform Independent

Java code compiles to bytecode that runs on any device with a JVM, enabling true cross-platform development.

üõ°Ô∏è

Secure

Built-in security features include bytecode verification, sandboxing, and no direct memory access, making Java ideal for networked applications.

üí™

Robust

Strong type checking, exception handling, and automatic garbage collection eliminate many common programming errors.

üîÑ

Multi-threaded

Native support for concurrent programming allows efficient utilization of modern multi-core processors.

üöÄ Write Once, Run Anywhere (WORA)

Java's most revolutionary concept is platform independence. Unlike languages like C++ that compile directly to machine code for a specific platform, Java takes a different approach:

üí° Key Insight

The JVM acts as an abstraction layer between your code and the underlying hardware. Each platform has its own JVM implementation that translates bytecode into native machine instructions, but your Java code remains the same across all platforms.

The WORA workflow

üîì public

An access modifier that makes the method accessible from anywhere. The JVM needs to call this method from outside the class, so it must be public.

üìå static

Means the method belongs to the class, not to instances. The JVM can call main() without creating an object of the class.

üö´ void

The return type indicating the method doesn't return any value. It just executes code and terminates.

üè† main

The required name for the entry point method. The JVM looks specifically for a method called "main" to start execution.

üìù String[] args

An array of strings that stores command-line arguments passed when running the program.

üì¶ class HelloWorld

Defines a class named HelloWorld. The filename must match the class name (HelloWorld.java).

public class ArithmeticOperators {

        x -= 3;   // x = x - 3 ‚Üí 12
        x *= 2;   // x = x * 2 ‚Üí 24
        x /= 4;   // x = x / 4 ‚Üí 6
        System.out.println("Final x = " + x);        // 6

public class LogicalOperators {

    public static void main(String[] args) {
        boolean a = true;
        boolean b = false;
        
        // Logical AND (both must be true)
        System.out.println("true && true: " + (true && true));   // true

        System.out.println("true || false: " + (a || b));       // true
        System.out.println("false || false: " + (b || b));      // false

        System.out.println("!true: " + !a);                      // false
        System.out.println("!false: " + !b);                     // true
        
        // Practical example
        int age = 25;
        boolean hasLicense = true;
        
        boolean canDrive = (age >= 18) && hasLicense;
        System.out.println("Can drive: " + canDrive);           // true
        
        boolean isWeekend = true;
        boolean isHoliday = false;
        boolean dayOff = isWeekend || isHoliday;
        System.out.println("Day off: " + dayOff);               // true
