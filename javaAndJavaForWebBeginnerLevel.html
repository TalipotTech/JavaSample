PART 1: CORE JAVA FUNDAMENTALS

Chapter 1: Getting Started

1.1 Detailed Concepts

The Java Ecosystem: Unlike languages that compile to machine code (like C++), Java compiles to Bytecode (.class files). This bytecode runs on the Java Virtual Machine (JVM). This is why Java is "Write Once, Run Anywhere"â€”Windows, Mac, and Linux all have their own JVMs that understand the same bytecode.

JDK vs. JRE:

  • JRE (Java Runtime Environment): Contains only what is needed to run Java programs (JVM + Libraries).
  • JDK (Java Development Kit): Contains the JRE plus tools like the compiler (javac) and debugger (jdb). As a developer, you must install the JDK.

Entry Point: Every Java application must have a class with a main method. This is the method the JVM looks for to start the program.

1.2 Sample Code

/**
 * This is a documentation comment.
 * Class name must match the file name (HelloWorld.java)
 */
public class HelloWorld {

    // 'public' means accessible from anywhere.
    // 'static' means we don't need to create an object of the class to run this.
    // 'void' means this method returns nothing.
    // 'String[] args' is an array of command-line arguments.
    public static void main(String[] args) {
        
        // Variables must declare a type (Strong Typing)
        String message = "Welcome to Java Mastery";
        int currentYear = 2023;
        boolean isJavaFun = true;

        // String Concatenation
        System.out.println(message + " - Year: " + currentYear);
        
        // Control Flow check
        if (isJavaFun) {
            System.out.println("Java is indeed fun!");
        }
    }
}

1.3 Common Pitfalls

  • Case Sensitivity: System.out.println works; system.out.println does not.
  • Missing Semicolons: Unlike Python or JavaScript, Java statements must end with a semicolon.

1.4 Practice Questions

QuestionAnswer

Chapter 2: Control Flow

2.1 Detailed Concepts

The switch Statement: A cleaner alternative to many if-else statements when checking a single variable against specific values.

Enhanced For-Loop (For-Each): Designed for iterating over arrays and collections without using an index counter.

2.2 Sample Code

public class FlowControl {
    public static void main(String[] args) {
        int dayOfWeek = 5;
        
        // Traditional Switch
        switch (dayOfWeek) {
            case 1:
                System.out.println("Monday");
                break; // Without break, it "falls through" to the next case!
            case 5:
                System.out.println("Friday - TGIF!");
                break;
            default:
                System.out.println("Mid-week day");
        }

        // Enhanced For-Loop
        String[] fruits = {"Apple", "Banana", "Cherry"};
        System.out.println("\nMy Fruit List:");
        for (String fruit : fruits) {
            System.out.println("- " + fruit);
        }
    }
}

2.4 Practice Questions

QuestionAnswer
What is the difference between x++ and ++x inside a loop condition?x++ (post-increment) returns the value of x first, then adds 1. ++x (pre-increment) adds 1 first, then returns the value.
Can a switch statement work on a String in Java?Yes, starting from Java 7.

Chapter 3: OOP Basics

3.1 Detailed Concepts

State vs. Behavior: Objects have State (fields/variables) and Behavior (methods).

The this keyword: Refers to the current instance of the object. It is used to distinguish between class fields and parameters with the same name (shadowing).

3.2 Sample Code

class Student {
    // Fields (State) - Encapsulated (private)
    private String name;
    private int age;

    // Constructor: Runs once when object is created
    public Student(String name, int age) {
        this.name = name; // 'this.name' is the field, 'name' is the parameter
        this.age = age;
    }

    // Method (Behavior)
    public void introduce() {
        System.out.println("Hi, I am " + name + " and I am " + age + " years old.");
    }
    
    // Getters (Accessors)
    public String getName() { return name; }
}

3.4 Practice Questions

QuestionAnswer
Can a class have more than one constructor?Yes, this is called Constructor Overloading.
What is a local variable vs. an instance variable?A local variable is declared inside a method and exists only while that method runs. An instance variable (field) is declared in the class and exists as long as the object exists.

Chapter 4: OOP Deep Dive

4.1 Detailed Concepts

Inheritance (extends): Allows code reuse. The subclass inherits all non-private members of the superclass.

Method Overriding: Redefining a parent's method in the child to provide specific behavior. Requires the @Override annotation.

Abstract Classes: Cannot be instantiated. They are designed to be subclassed. They can have both abstract (no body) and concrete methods.

4.2 Sample Code

// Abstract Parent Class
abstract class Shape {
    String color;

    public Shape(String color) {
        this.color = color;
    }

    // Abstract method: Child classes MUST implement this
    abstract double area();
    
    void displayColor() {
        System.out.println("Shape color is " + color);
    }
}

// Child Class
class Circle extends Shape {
    double radius;

    public Circle(String color, double radius) {
        super(color); // Calls parent constructor
        this.radius = radius;
    }

    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

Chapter 5: Arrays and Collections

5.1 Detailed Concepts

Wrapper Classes: Java Collections (List, Map) cannot store primitives (int, double). They store Objects. Java provides wrapper classes (Integer, Double) to wrap primitives into objects.

Generics (<>): Ensures type safety. Without generics, a List would accept any Object, leading to runtime crashes when casting.

5.2 Sample Code

import java.util.*;

public class CollectionDemo {
    public static void main(String[] args) {
        // 1. ArrayList - Dynamic array
        List<String> shoppingList = new ArrayList<>();
        shoppingList.add("Milk");
        shoppingList.add("Eggs");

        // 2. HashMap - Key/Value pairs
        Map<String, String> phoneBook = new HashMap<>();
        phoneBook.put("Alice", "555-1234");
        phoneBook.put("Bob", "555-9876");
    }
}

Chapter 6: Exception Handling & Basic I/O

6.1 Detailed Concepts

Checked vs. Unchecked Exceptions:

  • Checked: The compiler forces you to handle them (e.g., IOException). Usually recoverable.
  • Unchecked: Runtime errors (e.g., NullPointerException, ArithmeticException). Usually programming bugs.

Try-With-Resources: A syntax (Java 7+) that automatically closes resources (like Files) to prevent memory leaks.

6.2 Sample Code

import java.io.File;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class ExceptionDemo {
    public static void main(String[] args) {
        // Try-with-resources ensures 'scanner' closes automatically
        try (Scanner scanner = new Scanner(new File("numbers.txt"))) {
            while (scanner.hasNext()) {
                System.out.println(scanner.nextInt());
            }
        } catch (FileNotFoundException e) {
            System.err.println("File not found! Please create numbers.txt");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Chapter 7: Essential Java APIs

7.1 Detailed Concepts

String Immutability: Strings in Java are immutable. Every time you "edit" a string, Java creates a new String object.

StringBuilder: A mutable sequence of characters. Highly efficient for loops or string manipulation.

7.2 Sample Code

import java.time.LocalDate;

public class APIDemo {
    public static void main(String[] args) {
        // String Immutability
        String s1 = "Hello";
        String s2 = s1.concat(" World"); // Creates a new object!
        
        // StringBuilder (Efficient)
        StringBuilder sb = new StringBuilder();
        for(int i=0; i<100; i++) {
            sb.append(i); // Modifies the SAME object
        }

        // Java Time API
        LocalDate now = LocalDate.now();
        LocalDate future = now.plusDays(10);
    }
}

Chapter 10: JDBC

10.1 Detailed Concepts

PreparedStatement: Precompiles the SQL query. It is safer and faster than Statement. It prevents SQL Injection.

10.2 Sample Code

import java.sql.*;

public class JDBCDemo {
    static final String DB_URL = "jdbc:mysql://localhost/mydb";

    public static void main(String[] args) {
        String query = "SELECT id, name, salary FROM employees WHERE salary > ?";

        try (Connection conn = DriverManager.getConnection(DB_URL, "root", "pass");
             PreparedStatement pstmt = conn.prepareStatement(query)) {

            pstmt.setDouble(1, 50000.0);
            ResultSet rs = pstmt.executeQuery();

            while(rs.next()){
                System.out.println(rs.getInt("id"));
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

Chapter 15: Service Layer & Business Logic

15.1 Detailed Concepts

Separation of Concerns: Controller (HTTP), Service (Logic), Repository (SQL).

@Transactional: Ensures a group of DB operations either all succeed or all fail.

15.2 Sample Code

import org.springframework.stereotype.Service;

@Service
public class UserService {
    @Autowired
    private UserRepository userRepo;

    public User registerUser(User user) {
        if (user.getUsername() == null || user.getUsername().length() < 3) {
            throw new IllegalArgumentException("Username too short");
        }
        return userRepo.save(user);
    }
}

PART 3: CAPSTONE PROJECT

Final Project: Task Management System

Project Goal

Create a fully functional REST API where users can manage their daily tasks.

Requirements

  1. Setup: Spring Boot, H2 or MySQL.
  2. Entity: Task with id, title, description, completed.
  3. Repository: TaskRepository extending JpaRepository.
  4. Service: Logic to create, complete, and delete tasks.
  5. Controller: Endpoints for GET, POST, PUT, DELETE.

Solution Guide: Entity

import javax.persistence.*;

@Entity
public class Task {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false)
    private String title;
    
    private boolean completed = false; // Default false
    
    public Task(String title, String description) {
        this.title = title;
        this.description = description;
    }
    // Getters and Setters...
}

Solution Guide: Service Implementation

import org.springframework.stereotype.Service;

@Service
public class TaskService {
    @Autowired
    private TaskRepository taskRepository;

    public Task completeTask(Long id) {
        Task task = taskRepository.findById(id)
                .orElseThrow(() -> new RuntimeException("Task not found"));
        task.setCompleted(true);
        return taskRepository.save(task);
    }
}
