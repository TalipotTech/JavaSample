5.1

Inheritance Basics

Learn how classes can inherit properties and behaviors from other classes, promoting code reuse.

üß¨ What is Inheritance?

Inheritance is a fundamental OOP concept that allows a new class (child/subclass) to inherit attributes and methods from an existing class (parent/superclass). This promotes code reuse and establishes a natural hierarchy between classes.

üë®‚Äçüëß

Parent Class (Superclass)

The class whose properties and methods are inherited. Also called the base class.

üë∂

Child Class (Subclass)

The class that inherits from another class. Also called the derived class.

üîó

extends Keyword

The Java keyword used to establish inheritance between classes.

‚ôªÔ∏è

Code Reuse

Child classes automatically get all non-private members of the parent class.

Inheritance Syntax 
class ChildClass extends ParentClass {
    // Child class inherits all non-private members from ParentClass
    // Can add new fields and methods
    // Can override parent methods
}

5.2

The super Keyword

Access parent class constructors and methods using the super keyword.

‚¨ÜÔ∏è Understanding super

The super keyword refers to the parent class. It's used to call the parent's constructor and access parent class members that might be hidden by child class members.

üèóÔ∏è

super() Constructor

Calls the parent class constructor. Must be the first statement in child constructor.

üîß

super.method()

Calls a method from the parent class, useful when the method is overridden.

üì¶

super.field

Accesses a field from the parent class when it's shadowed by a child field.

Vehicle.java 
// Parent class
public class Vehicle {
    protected String brand;
    protected String model;
    protected int year;
    protected double price;
    
    // Constructor
    public Vehicle(String brand, String model, int year, double price) {
        this.brand = brand;
        this.model = model;
        this.year = year;
        this.price = price;
        System.out.println("Vehicle constructor called");
    }
    
    public void start() {
        System.out.println("Vehicle is starting...");
    }
    
    public void displayInfo() {
        System.out.println("Brand: " + brand);
        System.out.println("Model: " + model);
        System.out.println("Year: " + year);
        System.out.println("Price: $" + price);
    }
}
Car.java 
// Child class demonstrating super keyword usage
public class Car extends Vehicle {
    private int numDoors;
    private String fuelType;
    private boolean isAutomatic;
    
    // Constructor using super() to call parent constructor
    public Car(String brand, String model, int year, double price, 
              int numDoors, String fuelType, boolean isAutomatic) {
        
        // super() MUST be the first statement!
        super(brand, model, year, price);
        
        this.numDoors = numDoors;
        this.fuelType = fuelType;
        this.isAutomatic = isAutomatic;
        System.out.println("Car constructor called");
    }
    
    // Override start() but also call parent's start()
    @Override
    public void start() {
        super.start();  // Call parent's start method
        System.out.println("Car engine is running!");
        System.out.println("Dashboard lights are on.");
    }
    
    // Override displayInfo() and extend it
    @Override
    public void displayInfo() {
        super.displayInfo();  // Call parent's displayInfo
        System.out.println("Doors: " + numDoors);
        System.out.println("Fuel Type: " + fuelType);
        System.out.println("Transmission: " + (isAutomatic ? "Automatic" : "Manual"));
    }
    
    // Car-specific methods
    public void honk() {
        System.out.println("Beep beep!");
    }
}

super() Must Be First!

When calling the parent constructor with super(), it must be the very first statement in the child constructor. Java enforces this rule strictly.

üîó Constructor Chaining

When you create an object of a child class, the parent constructor is always called first, either explicitly or implicitly. This creates a chain of constructor calls.

ConstructorChaining.java 
// Demonstrating constructor chaining

class Grandparent {
    public Grandparent() {
        System.out.println("1. Grandparent constructor");
    }
}

class Parent extends Grandparent {
    public Parent() {
        // super() is called implicitly here
        System.out.println("2. Parent constructor");
    }
}

class Child extends Parent {
    public Child() {
        // super() is called implicitly here
        System.out.println("3. Child constructor");
    }
}

public class ConstructorChaining {
    public static void main(String[] args) {
        System.out.println("Creating a Child object:");
        Child child = new Child();
    }
}

1. Grandparent constructor
2. Parent constructor
3. Child constructor
üí°

Implicit super() Call

If you don't explicitly call super(), Java automatically inserts super() (the no-argument version) as the first line. This fails if the parent class doesn't have a no-arg constructor!

5.3

Method Overriding

Learn how child classes can provide their own implementation of parent methods.

üîÑ What is Method Overriding?

Method overriding occurs when a child class provides a specific implementation for a method that is already defined in its parent class. The overriding method must have the same signature (name, parameters, and return type).

Override Signature Requirements

@Override public void methodName (params)
@Override annotation (recommended)
Same or wider access modifier
Same return type
Same method name
Same parameters

// Parent class with methods to override
public class Shape {
    protected String name;
    protected String color;

        this.name = name;
        this.color = color;
    }

        return 0.0;

    public double calculatePerimeter() {
        return 0.0;

    public void display() {

}

Always Use @Override

The @Override annotation is optional but highly recommended. It tells the compiler you intend to override a method, and it will give an error if the method doesn't actually override anything (e.g., due to a typo).

Overriding vs Overloading

Aspect Method Overriding Method Overloading
Definition Same method in child class Same method name, different parameters
Classes Requires inheritance (two classes) Can be in the same class
Parameters Must be the same Must be different

5.4

Polymorphism

Understand how objects can take many forms and behave differently based on their actual type.

üé≠ What is Polymorphism?

Polymorphism means "many forms." In Java, it allows objects of different classes to be treated as objects of a common parent class. The actual method that gets called is determined at runtime based on the object's actual type, not the reference type.

üìö

Compile-Time Polymorphism

Method overloading - resolved during compilation based on method signature.

‚ö°

Runtime Polymorphism

Method overriding - resolved during execution based on actual object type.

üîç Understanding Reference vs Object Types

Reference Type
Animal
=
new
Object Type
Dog()

The reference type determines what methods you can call. The object type determines which version runs.

PolymorphismDemo.java 
// Base class
class Animal {
    public void makeSound() {
        System.out.println("Some generic animal sound");
    }
    
    public void eat() {
        System.out.println("Animal is eating");
    }
}

// Child classes with overridden methods
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Woof! Woof!");
    }
    
    public void fetch() {
        System.out.println("Dog is fetching the ball");
    }
}

class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Meow! Meow!");
    }
    
    public void scratch() {
        System.out.println("Cat is scratching");
    }
}

class Bird extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Tweet! Tweet!");
    }
    
    public void fly() {
        System.out.println("Bird is flying");
    }
}

public class PolymorphismDemo {
    public static void main(String[] args) {
        // Polymorphic references - parent type, child object
        Animal animal1 = new Dog();
        Animal animal2 = new Cat();
        Animal animal3 = new Bird();
        
        System.out.println("=== Polymorphism in Action ===");
        // Same method call, different behavior!
        animal1.makeSound();  // Calls Dog's makeSound
        animal2.makeSound();  // Calls Cat's makeSound
        animal3.makeSound();  // Calls Bird's makeSound
        
        System.out.println("\n=== Using an Array of Animals ===");
        Animal[] animals = {new Dog(), new Cat(), new Bird()};
        
        for (Animal animal : animals) {
            animal.makeSound();  // Polymorphic method call
            animal.eat();        // Inherited method
            System.out.println();
        }
    }
}

üîÑ instanceof and Type Casting

Sometimes you need to check the actual type of an object or cast it to access child-specific methods.

TypeCastingDemo.java 
public class TypeCastingDemo {
    public static void main(String[] args) {
        // Array of different animals
        Animal[] animals = {
            new Dog(),
            new Cat(),
            new Bird(),
            new Dog()
        };
        
        for (Animal animal : animals) {
            // Check the actual type using instanceof
            if (animal instanceof Dog) {
                System.out.println("Found a dog!");
                Dog dog = (Dog) animal;  // Downcast to access Dog methods
                dog.fetch();
            } else if (animal instanceof Cat) {
                System.out.println("Found a cat!");
                Cat cat = (Cat) animal;  // Downcast to access Cat methods
                cat.scratch();
            } else if (animal instanceof Bird) {
                System.out.println("Found a bird!");
                Bird bird = (Bird) animal;  // Downcast to access Bird methods
                bird.fly();
            }
            System.out.println();
        }
        
        // Java 16+ Pattern Matching (cleaner syntax)
        System.out.println("=== Pattern Matching (Java 16+) ===");
        Animal myAnimal = new Dog();
        
        if (myAnimal instanceof Dog d) {
            d.fetch();  // d is already cast to Dog
        }
    }
}

5.5

Abstract Classes

Create blueprints that define structure but leave implementation to child classes.

üìã What are Abstract Classes?

An abstract class is a class that cannot be instantiated directly. It serves as a base class that defines common behavior and can include both complete methods and abstract methods (methods without a body) that child classes must implement.

üö´

Cannot Instantiate

You cannot create objects directly from an abstract class using new.

üìù

Abstract Methods

Methods declared without a body - child classes must provide implementation.

‚úÖ

Concrete Methods

Can also have regular methods with full implementations.

üîß

Constructors

Can have constructors that are called by child class constructors.

Abstract Class Syntax 
abstract class ClassName {
    // Fields
    protected int field;
    
    // Constructor
    public ClassName(int field) {
        this.field = field;
    }
    
    // Abstract method (no body) - MUST be implemented by child
    public abstract void abstractMethod();
    
    // Concrete method (has body) - inherited as-is
    public void concreteMethod() {
        System.out.println("Regular method");
    }
}
Employee.java 
// Abstract class - cannot be instantiated
public abstract class Employee {
    protected String name;
    protected int employeeId;
    protected double baseSalary;
    
    // Constructor
    public Employee(String name, int employeeId, double baseSalary) {
        this.name = name;
        this.employeeId = employeeId;
        this.baseSalary = baseSalary;
    }
    
    // Abstract method - each employee type calculates salary differently
    public abstract double calculateSalary();
    
    // Abstract method - each employee type has different responsibilities
    public abstract String getResponsibilities();
    
    // Concrete method - same for all employees
    public void displayInfo() {
        System.out.println("Employee ID: " + employeeId);
        System.out.println("Name: " + name);
        System.out.println("Base Salary: $" + baseSalary);
        System.out.printf("Total Salary: $%.2f%n", calculateSalary());
        System.out.println("Responsibilities: " + getResponsibilities());
    }
    
    // Concrete method
    public void clockIn() {
        System.out.println(name + " has clocked in.");
    }
}

When to Use Abstract Classes

Use abstract classes when you want to share code among several closely related classes, when you expect classes that extend your abstract class to have many common methods or fields, or when you need non-public members or non-static/non-final fields.

5.6

Interfaces

Define contracts that classes must follow, enabling multiple inheritance of type.

üìú What are Interfaces?

An interface is a completely abstract type that defines a contract of methods that implementing classes must provide. Unlike classes, a class can implement multiple interfaces, enabling a form of multiple inheritance.

üìù

Abstract Methods

All methods are implicitly public and abstract (before Java 8).

üîí

Constants Only

Fields are implicitly public, static, and final (constants).

üîó

Multiple Implementation

A class can implement multiple interfaces.

‚öôÔ∏è

Default Methods (Java 8+)

Interfaces can have methods with default implementations.

Interface Syntax 
interface InterfaceName {
    // Constants (public static final by default)
    int CONSTANT = 100;
    
    // Abstract methods (public abstract by default)
    void abstractMethod();
    String anotherMethod(int param);
    
    // Default method (Java 8+)
    default void defaultMethod() {
        System.out.println("Default implementation");
    }
    
    // Static method (Java 8+)
    static void staticMethod() {
        System.out.println("Static method in interface");
    }
}
Interfaces.java 
// Interface definitions
interface Drawable {
    void draw();
    String getColor();
}

interface Resizable {
    void resize(double factor);
    double getSize();
}

interface Movable {
    void move(int x, int y);
    int getX();
    int getY();
    
    // Default method
    default void moveToOrigin() {
        move(0, 0);
    }
}
GraphicCircle.java 
// Class implementing multiple interfaces
public class GraphicCircle implements Drawable, Resizable, Movable {
    private String color;
    private double radius;
    private int x, y;
    
    public GraphicCircle(String color, double radius, int x, int y) {
        this.color = color;
        this.radius = radius;
        this.x = x;
        this.y = y;
    }
    
    // Implementing Drawable interface
    @Override
    public void draw() {
        System.out.println("Drawing a " + color + " circle at (" + x + ", " + y + ")");
        System.out.println("Radius: " + radius);
    }
    
    @Override
    public String getColor() {
        return color;
    }
    
    // Implementing Resizable interface
    @Override
    public void resize(double factor) {
        radius *= factor;
        System.out.println("Circle resized. New radius: " + radius);
    }
    
    @Override
    public double getSize() {
        return radius;
    }
    
    // Implementing Movable interface
    @Override
    public void move(int newX, int newY) {
        this.x = newX;
        this.y = newY;
        System.out.println("Circle moved to (" + x + ", " + y + ")");
    }
    
    @Override
    public int getX() { return x; }
    
    @Override
    public int getY() { return y; }
    
    // Uses default method from Movable interface
}

        circle.move(50, 50);
        circle.moveToOrigin();  // Default method

        // Can reference object through any implemented interface

When to Use Interfaces

Use interfaces when unrelated classes need to implement the same behavior (like Comparable), when you want to specify behavior but not implementation, or when you need to achieve multiple inheritance of type.

Let's see a practical example using interfaces for a payment processing system:

PaymentSystem.java 
// Payment interface
interface PaymentMethod {
    boolean processPayment(double amount);
    String getPaymentType();
    
    default void printReceipt(double amount) {
        System.out.println("Payment of $" + amount + " processed via " + getPaymentType());
    }
}

// Credit Card implementation
class CreditCard implements PaymentMethod {
    private String cardNumber;
    private String cardHolder;
    
    public CreditCard(String cardNumber, String cardHolder) {
        this.cardNumber = cardNumber;
        this.cardHolder = cardHolder;
    }
    
    @Override
    public boolean processPayment(double amount) {
        System.out.println("Processing credit card payment...");
        System.out.println("Card: **** **** **** " + cardNumber.substring(12));
        return true;
    }
    
    @Override
    public String getPaymentType() {
        return "Credit Card";
    }
}

// PayPal implementation
class PayPal implements PaymentMethod {
    private String email;
    
    public PayPal(String email) {
        this.email = email;
    }
    
    @Override
    public boolean processPayment(double amount) {
        System.out.println("Processing PayPal payment...");
        System.out.println("Account: " + email);
        return true;
    }
    
    @Override
    public String getPaymentType() {
        return "PayPal";
    }
}

// Cryptocurrency implementation
class Cryptocurrency implements PaymentMethod {
    private String walletAddress;
    private String cryptoType;
    
    public Cryptocurrency(String walletAddress, String cryptoType) {
        this.walletAddress = walletAddress;
        this.cryptoType = cryptoType;
    }
    
    @Override
    public boolean processPayment(double amount) {
        System.out.println("Processing " + cryptoType + " payment...");
        System.out.println("Wallet: " + walletAddress.substring(0, 10) + "...");
        return true;
    }
    
    @Override
    public String getPaymentType() {
        return cryptoType;
    }
}

public class PaymentSystem {
    public static void main(String[] args) {
        // Array of different payment methods (polymorphism!)
        PaymentMethod[] payments = {
            new CreditCard("1234567890123456", "John Doe"),
            new PayPal("john@example.com"),
            new Cryptocurrency("0x1234abcd5678efgh", "Bitcoin")
        };
        
        double[] amounts = {99.99, 49.50, 0.005};
        
        for (int i = 0; i < payments.length; i++) {
            System.out.println("=== Payment " + (i + 1) + " ===");
            
            if (payments[i].processPayment(amounts[i])) {
                payments[i].printReceipt(amounts[i]);  // Default method
            }
            System.out.println();
        }
    }
}
